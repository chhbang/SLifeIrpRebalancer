# 연금나침반 v1.0.5

삼성생명 IRP(개인형 퇴직연금) 계좌의 펀드/원리금보장형 비중 조정을 AI(Claude·Gemini·GPT)의 도움으로 계획하기 위한 Windows 데스크톱 도구입니다.

> ⚠️ **본 앱은 삼성생명보험주식회사와 무관한 비공식(third-party) 도구이며, AI가 생성한 제안은 참고용 정보일 뿐 투자 권유가 아닙니다.** 모든 투자 결정과 그 결과에 대한 책임은 사용자 본인에게 있습니다.

## v1.0.5 변경 사항

### 새 기능: 리밸런싱 이력 보관 + 직전 회차 컨텍스트 자동 첨부

분기에 한 번씩 리밸런싱을 받을 때, **AI가 이전 회차의 추천을 알고서 새 추천을 작성**하게 만들어 일관성 있는 장기 전략을 유지할 수 있게 했습니다.

- 좌측 메뉴에 **"이력"** 화면 추가 — 저장한 회차들을 시간순으로 나열, 그때의 markdown을 다시 보고 PDF로 재내보내기 / 삭제 / 다음 리밸런싱에 참고하기 가능
- AI 리밸런싱 화면에 **"이전 회차 참고"** 콤보 추가 — 선택하면 새 프롬프트의 "## 직전 리밸런싱 회차 (참고용)" 섹션에 그때의 시점·AI 모델·추가 요구사항·추천 markdown 원문이 자동으로 들어갑니다. AI에게는 "그때 추천을 단순 답습하지 말고 현 시점에서 다시 검토하라"는 가이드도 함께 전달됩니다
- AI 리밸런싱 화면에 **"이력에 저장"** 버튼 추가 — 한 회차에서 여러 AI/모델을 시도해 보고 마음에 드는 결과만 명시적으로 저장하는 정책 (자동 저장 안 함)

### 새 기능: 사용자 클라우드 폴더 연동 (다중 PC 동기)

Google Drive 데스크톱 / OneDrive / Dropbox 등 **사용자가 이미 쓰는 클라우드 동기 폴더**를 그대로 활용해 두 PC 사이에 IRP 데이터가 자동으로 흐르게 했습니다. 별도 OAuth/로그인 없이 사용자의 기존 클라우드 클라이언트가 동기를 처리합니다.

- 환경 설정에 **"동기화 폴더"** 입력 추가. 비워 두면 기존처럼 LocalState만 사용
- 폴더 지정 시 자동 동기되는 데이터:
  - 계좌 정보 (`account.json`)
  - 상품 카탈로그 (`catalog.json`)
  - 리밸런싱 이력 (`History\<timestamp>_<provider>.json`)
- 절대 동기 폴더에 기록되지 않는 데이터:
  - **API Key** (Windows 자격증명 보관소 그대로)
  - **임포트한 원본 HTML** (가입자 성명·계좌번호 포함되어 있어 처음부터 저장 안 함)
- LocalState와 동기 폴더가 모두 있으면 **mtime이 더 새로운 쪽**을 따라가므로 PC1 저장 → 클라우드 → PC2 자동 픽업
- 동기 폴더에는 보유 잔고·수익률 등 금융 정보가 들어가니 **본인만 접근 가능한 위치**(개인 OneDrive 등)에 지정해 주세요

## 설치 방법

다음 두 파일을 함께 받으세요:

| 파일 | 용도 |
| --- | --- |
| `PensionCompass_1.0.5.0_x64.msixbundle` | 앱 본체 |
| `PensionCompass_1.0.5.0_x64.cer` | self-signed 코드 서명 인증서 |

### 1단계: 인증서 신뢰 등록 (PC당 최초 1회)

자체 서명 인증서이므로 받는 PC가 발급자(CN=Changhwan Bang)를 신뢰하도록 등록해야 합니다.

1. `PensionCompass_1.0.5.0_x64.cer` 더블클릭
2. **인증서 설치(I)** 버튼 클릭
3. 저장소 위치: **로컬 컴퓨터(L)** 선택 → 다음
4. 인증서 저장소: **모든 인증서를 다음 저장소에 저장(P)** → 찾아보기 → **신뢰할 수 있는 사람** 선택 → 확인 → 다음 → 마침

### 2단계: 앱 설치

`PensionCompass_1.0.5.0_x64.msixbundle` 더블클릭 → "설치" 버튼.

설치 완료 후 시작 메뉴에서 "연금나침반"으로 검색해 실행할 수 있습니다.

### 업데이트하는 경우

기존 버전이 이미 설치되어 있다면 새 .msixbundle을 더블클릭하면 in-place 업데이트됩니다 (인증서 재등록 불필요, 기존 입력 데이터 유지). 동기화 폴더 설정도 그대로 유지됩니다.

## 시스템 요구 사항

- Windows 10 17763(1809) 이상 / Windows 11
- x64 아키텍처
- AI 호출용 인터넷 연결 + 사용자 본인의 API Key (Anthropic Claude / Google Gemini / OpenAI 중 하나 이상)

## 데이터 보관

- **계좌·카탈로그·이력**: 사용자 PC의 앱 LocalState 폴더 + (지정 시) 동기화 폴더 양쪽에 미러 저장. JSON 평문이며 사용자 본인의 PC/클라우드 계정 안에서만 보입니다
- **API Key 3개**: Windows 자격증명 보관소 (사용자 계정으로 암호화). 동기화 폴더에 절대 기록되지 않음
- **AI 호출 시**: 보유 현황과 카탈로그가 사용자가 선택한 AI 공급자(Anthropic / Google / OpenAI)에 직접 전송됩니다. 그 외 외부 전송은 없습니다
